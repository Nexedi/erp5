• Components

    A Set of Basic Layout Principles and Examples.

  Page Structure

Page Header

  • Single page web applications use a fixed header and content section. Fixed footers should be avoided due to performance issues (flickering, blocking, repositioning).

    The header will always consist of a main element (title, image, logo, breadcrumb) and optional button menus with a maximum of three buttons. The main element may be placed in the middle with button menus right and left.

  • Alternatively the main element (title, image, logo, breadcrumb) is placed on the left with a single button group on the opposite side. Note that buttons should be responsive and change to icon-only on small devices.

    This layout would allow for a search form inside the header.

Page Subheader

  • A header can have an subheader with a maximum of five buttons. This fast-access navigation bar should display icons along with text for all screen resolutions.

Panels

  • Panels can be used from either side. A left hand panel should always provide links to key sections of the application (navigation) as well as pages, that are used not frequently enough to warrant putting them within direct reach (for example logout, settings menus).

    On the other hand, a right hand panel should aid the user in performing tasks, like filling out a form or selecting search/filtering criteria.

    Panels should differentiate in color and contain a usage specific header to indicate their purpos. The panel header should always include a close button to hide the panel.

  Content Structure

Headers/Content

  • Content sections may contain four types of headers:

    • A fullwidth header (immideately following and contrasting the page header) containing title and optional subtitle and vis-a-vis content menu.

    • A section header - an introductory element containing title plus optional subtitle and content menu. The section header should use one of the gradient backgrounds or a background image.

    • A plain header made of an icon and title on plain background which can be used repeatedly throughout a content section for highlighting and structuring the layout.

    • A inline header used inside a content section containing just a title.

    Page content itself is set in content-sections, which adapt to the respective header used (fullwidth, section, plain).

  Layout

Grid

  • Content should always be structured in a grid that expands from single to multi columns with growing screen size. The grid can be used within a section element or as a standalone grid.

  Themes

Palette

  • A theme should consist of four colors (dark, medium, light hue and text color). For most web applications, a colored, constrasting/accent, dark and light theme should be sufficient.

    Themes should provide two solid tones with one also available as linear and/or radial gradient.

  Tables

Basic Table

  • Display tabular data on large displays should be column based while smartphone should display a list with each table row becoming a list element. This avoids having to deal with column priorities and heavy DOM manipulations.

    Tables should provide an associated toolbar as well as controls for pagination and filter/sorting of tabular data.

Filter/Sorting/Pagination

  • Similar to list controls a table should include controls to paginate, filter, search and sort data.

    Pagination should only be used on large screen sizes switching to endless scroll loading of new records.

  • Search, Filter and Sorting should share syntax and UI components. Filters are advanced search queries, which can be constructed and saved through a filter menu (not pictured). Alternatively the syntax can be entered directly into the search field along with plain (fulltext) searches.

    All active criteria should be visible as tags either directly on the page or in the respective menu (filtermenu > show active/existing filters, sorting > show active/existing sorting criteria). Tags are editable by clicking on them. This will load the underlying query into the search field. By default all tags can be deleted. Filters entered directly into the search input can be saved.

  Forms

Form Basics

  • Forms are divided into half-screen and full-screen containers. On small screen sizes all containers switch to fullscreen allowing with additional separators to for spacings.

    Labels are vis-a-vis the respective input when sufficient space is available. On small screens they switch to being positioned above the input field leaving enough space for validation messages. Labels and legends should be distinctive from plain text, either by decreasing font-size, using italic or a different letter-spacing.

  • Fields should leave space for a right-hand input button, for example to invoke a date-selector gadget or apply filters to search inputs.

    It should be differentiated between disabled and readonly fields, the latter for example using transparency in addition to rendering the input inaccessible.

  List Elements

List Basics

  • List items can be used in multiple areas (Grids, Cards, Lists).

    When working with plain lists, any of the following combinations of content can be used with or without a link:

    • Divider (text only)

    • Headline only

    • Headline with subtext

    • Headline (with subtext) with image

    • Headline (with subtext) with icon (left)

    • Headline (with subtext) with custom icon (right)

    • All of the above with bubble counter
