Commandments

Opinionated Guidelines for Developing Web Applications.

Email Download

1. Smallest Size First

- Applications should support multiple devices (smartphone, tablet, desktop, ...) but be designed for smallest size first -

2. Single Codebase

- A single codebase should tailor to all devices/platforms -

3. Interaction over Typing

- The smaller the device the more cumbersome typing is. Enforce interactions vs requiring users to type -

4. Usability over Functionality

- Ease of use dictatetes application design, not functionalities provided -

5. Embrace Path of Least Interaction

- The less interactions are necessary to complete an action, the better -

6. Embrace Interaction Flow

- Interaction elements are positioned within the "flow" of application usage (eg. form save after form) -

7. Key Interactions Visible without Scrolling

- The key interactions of a page are accessible without the user having to scroll -

8. Highlight Key Interaction

- The key interaction is highlighted for ease of navigation -

9. Animate Only If Possible

- Use animations only if they work. Partially working animations ruin look & feel!! -

10. Four Themes: Light + Dark + Color + Active

- Each theme with a main and light color (total eight colors) -

11. One Font, Four Sizes, Two Weights

- Applications use a single font in up to four sizes and two weights -

12. Max Two breakpoints per device

- One or two breakpoints per device (smartphone, tablet, desktop, ...) must suffice -

13. Flat Single Color Icons

- Iconsets must be single color, flat icons -
