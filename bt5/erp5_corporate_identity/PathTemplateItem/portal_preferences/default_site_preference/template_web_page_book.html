Synthesis

This should be the summary of the document, for example: this document is an example of a erp5 book document. It is usable for project reports, sale offers and sale orders. It's plain HTML and can be rendered as html (for displaying on websites) or as pdf using the export dialog. The following options can be activated through the export dialog:

  • Add table of content
  • Add table of history (version, signatures, distribution - XXX only raw)
  • Add table of references (abbreviations, referenced/applicable documents, figures, tables), applies to links in []
  • Embed (internal) documents, applies to links with references
  • Embed (internal) reports, applies to links with url parameter report

After this you can write plain html and embed documents as needed. Some things to note:

Headers

Make sure to number headers correctly (h1->h2->h3...). No numbering (>h1<1....) is needed. Also, no html inside headers is allowed.

Links

Applicable Documents

Look like this [AD]. Note you need to wrap in [] and add "AD", href and title. Semicolons in titles fill the different cells in the generated table.

Referenced Documents

Same as applicable documents, so [RD], note that doubles are counted correctly, [AD] and everything is auto-numbered and linked from the tables.

Abbreviations

Have a similar synatx, [ERP], with the only difference being the abbreviation instead of AD/RD

Plain Links

Plain links will not be formatted, unless embedding documents/reports is active. Embedding documents will cause internal (!) links to be replaced with the content of the document. Reports

Reports are not part of the standard template, so they have to be implemented indivually in template customizations. Calling a nonexisting report will not throw an error and only leave the link untouched.

To embedding a report just create a link to the module and object (no references!, report must be callable on the object) and specify the report name and subsequent report parameters. For example:

Project Reports

Line Breakdown Report

Sale Order Reports

Cost Effort Report
Deliverable Report
Project Breakdown
Requirement Coverage
Requirement Report

These reports are part of the Nexedi corporate identity template and will be rendered if the template is installed. Else, only the links will be visible.
