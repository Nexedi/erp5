OfficeJS Preliminary Specification

Version: 001 - 2013-11-19

This documents defines the specification of OfficeJS offline HTML5 applications.

Dependencies

OfficeJS applications depends on the following libraries:

  • jQuery Mobile: platform independent user interface widgets (more than 30 Web browsers supported)
  • J-IO: backend independent persistent data management (offline, online, replication, encryption, etc.)
  • RenderJS: reusable component system (optional)

Applications submitted to OfficeJS HTML5 appstore must provide support for these libraries based on the following rules:

jQuery Mobile JIO RenderJS
Application has no persistent data recommended optional optional
Application has persistent data recommended required optional
Application data can be embedded by another application recommended required required

Structure

An OfficeJS application is an HTML5 file that defines metadata and dependencies (CSS, JS) of the application. You can use the OfficeJS application template below to start.

<!DOCTYPE html>
<html manifest="manifest.appcache">
  <head>
    <meta http-equiv="Content-type" content="text/html; charset=utf-8">
    <meta name="viewport" content="width=device-width, user-scalable=no" />

    <!-- required meta tags/information -->
    <title>Application Title</title>
    <meta name="description" content="Application Description"/>
    <meta name="keywords" content="Category_1, Category_2">
    <meta name="author" content="John Doe|johndoe@internetl.com"/>

    <!-- CSS -->
    <link rel="stylesheet" href="http://code.jquery.com/mobile/git/jquery.mobile-git.css">

    <!-- Custom Stylesheets -->
    <link rel="stylesheet" href="path/to/stylesheet_a">
    
    <!-- JQUERY/JQUERY MOBILE -->
    <script type="text/javascript" src="http://code.jquery.com/jquery-1.10.1.js"></script>
    <script type="text/javascript" src="http://code.jquery.com/mobile/git/jquery.mobile-git.js"></script>

    <!-- JIO -->
    <script type="text/javascript" src="path/to/sha256.amd.js"></script>
    <script type="text/javascript" src="path/to/rsvp-custom.js"></script>
    <script type="text/javascript" src="path/to/jio.js"></script>
    <script type="text/javascript" src="path/to/complex_queries.js"></script>
    <!-- JIO storages, for example -->
    <script type="text/javascript" src="path/to/localstorage.js"></script>

    <!-- RenderJS -->
    <script type="text/javascript" src="path/to/renderjs.js"></script>

    <!-- 3rd Party Plugins/Custom Scripts -->
    <script type="text/javascript" src="path/to/plugin_a.js"></script>
    <script type="text/javascript" src="path/to/plugin_b.js"></script>
  
  </head>
  <body>
    <!-- Content of application -->
  </body>

</html>

Please note:

  • Applications should have a manifest file specifying files to be cached
  • Custom script and additional plugins can simply be added at the end of the HTML5 file's body. Examples of easy to integrate plugins include:
    • Form Validation - Validval
    • i18n Support - i18next
  • Beware of the jQuery Mobile Ajax based navigation when creating an HTML5 application.
